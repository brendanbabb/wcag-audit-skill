Four audit modes

Scan. Batch. Compare. Track.

From a single-page deep dive to site-wide compliance tracking, each mode is optimized for tool call efficiency.

01 — Single page

Deep audit

Full 3-source audit of one URL. Accessibility tree analysis reveals what axe-core misses — unlabeled navs, heading misuse, phantom focusable elements, structural issues.

Key output: Interactive dashboard with severity-filtered issue cards, element-level detail from the a11y tree, and specific fix suggestions.
02 — Batch crawl

Site scorecard

Audit 5–10 pages in one run. Uses the combined audit script — ONE tool call per page after navigation. Identifies template-level vs page-specific issues automatically.

Key output: Rolled-up scorecard with per-page scores, template-level issue cards, and a horizontal bar chart sorted by score.
03 — Before/after diff

Remediation progress

Compare two audit runs to measure what was fixed. Shows resolved, new, persisting, and improved issues with score delta. Generates leadership-ready narrative summaries.

Key output: Progress report showing "7 of 19 issues resolved, score improved 58 → 78" — ready to paste into an email.
04 — Tracking spreadsheet

XLSX export

Export any audit to a structured spreadsheet with score summary, all issues, template-level issues, and audit metadata. Designed for longitudinal tracking and team handoff.

Key output: 4-sheet XLSX with conditional formatting, ready for leadership reporting and compliance tracking.
Three data sources

axe-core. Accessibility tree. Manual inspection.

Each source catches issues the others miss. Single-page mode uses all three; batch mode uses axe-core + manual checks for efficiency.

Automated scan

axe-core 4.8.4

Industry-standard accessibility engine injected into the live page. Tests against WCAG 2.0 A/AA, WCAG 2.1 A/AA, and best-practice rules. Catches color contrast, missing labels, ARIA misuse, list structure violations.

Catches: Color contrast ratios, form label associations, ARIA attribute validation, list structure, link purpose.
Structural analysis

Chrome accessibility tree

The browser's accessibility tree represents what assistive technology actually sees. Reveals heading hierarchy gaps, unlabeled navigation regions, phantom focusable elements, and structural issues invisible to automated scans.

Catches: 247-element Google Translate widget noise, zero-width-space ghost links, heading misuse (hours styled as H5), unlabeled nav identification.
DOM inspection

Manual checks

JavaScript checks for things neither axe-core nor the a11y tree surface cleanly: skip links, lang attributes, images without alt (vs empty alt), external links opening new tabs without warnings, positive tabindex.

Catches: Skip link presence, lang attribute, alt vs empty alt distinction, target="_blank" without warnings, tabindex disruptions.

Built for tool call budgets

Every Claude in Chrome tool call counts against a per-conversation limit. This skill is designed to maximize audit coverage within those constraints.

Key innovation: Combined audit script

One javascript_tool call loads axe-core, waits for it, runs the WCAG scan, executes all manual DOM checks, and returns combined JSON results. This replaces the 4–5 separate tool calls per page that the original version required.

Platform awareness

SharePoint-ready

Built and tested on SharePoint 2013/2016/2019 government sites. The skill understands common SharePoint patterns and filters noise from real issues.

Template-level detection

Automatically identifies issues that appear on 80%+ of pages — master page fixes that improve every page at once. Separates them from page-specific content issues in the report.

SharePoint noise filtering

Recognizes empty web part zones (PageContent2-11), Google Translate widget clutter (247 generic elements), aria-labelledby on divs with no ARIA role, and other SharePoint artifacts.

Get started

Start auditing in three steps

Setup
1

Download the skill files

Get SKILL.md, references/axe-core-scripts.md, and references/wcag-quick-ref.md from GitHub.

2

Upload to Claude

Add the skill to your Claude.ai environment via Settings → Features → Skills. Or copy the directory into your Claude Code skill folder.

3

Connect Claude in Chrome

Install the Claude in Chrome extension for full 3-source audits. Without it, the skill falls back to partial audits via web_fetch.

Design principles

Built for real government work

Efficient

Every audit mode is designed around tool call budgets. Combined scripts, no wasted wait calls, optional depth levels for different page counts.

Actionable

Every issue includes a specific fix suggestion — actual code changes, not vague advice. Template vs page scope identified so teams know where to fix.

Multi-source

Three complementary audit sources catch different classes of issues. Multi-source findings are flagged as higher confidence.

Transparent

Every report notes methodology limitations. Automated tools catch 30-40% of WCAG issues — the skill says so, every time.

Shareable

XLSX exports, Visualizer dashboards, and leadership-ready narrative summaries. Built for teams, not just the person running the audit.
